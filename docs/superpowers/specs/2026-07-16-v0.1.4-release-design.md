# v0.1.4 发布设计

## 背景

Task 161-165 累积了 5 项未发布变更，覆盖单模型元数据检索、Response 完整生命周期（读取+删除）和两项锁优化。按项目版本发布节奏，应发布 v0.1.4。

## 发布范围

- 单模型元数据检索 `GET /v1/models/{model}`（Task 163）
- Response 读取 `GET/HEAD /v1/responses/{response_id}`（Task 164）
- Response 删除 `DELETE /v1/responses/{response_id}`（Task 165）
- response store 锁持有时间优化（Task 161）
- provider health 读锁优化（Task 162）

## 版本策略

- 遵循 SemVer，所有变更为向后兼容新增，递增 PATCH 版本：0.1.3 → 0.1.4。
- 版本号 0.1.4 注入到二进制和 Docker 镜像中。
- Changelog 日期为 2026-07-16。

## 发布流程

1. 将 Unreleased changelog 条目移动到 `## 0.1.4 - 2026-07-16`。
2. 更新 `docs/release.md` 中的示例版本号。
3. 创建 Task 166 记录和 plan/design 文档。
4. 在 WSL `Ubuntu-24.04` 中运行 `make release-check VERSION=0.1.4`。
5. 提交发布变更到 master。
6. 创建 annotated tag `v0.1.4` 并推送 `origin/master` 和 `v0.1.4`。

## 非目标

- 不推送 Docker 镜像（由后续任务补充）。
- 不自动创建 GitHub/Gitee release。
- 不包含新功能实现。
