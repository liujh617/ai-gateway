# v0.1.2 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this release plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Do not delegate tag creation or push to a subagent.

**Goal:** 发布 `open-ai-gateway` v0.1.2，包含 race 测试稳定性、Azure 本地 smoke 和 shell LF 约束，并确保 tag 精确指向通过发布验证的提交。

**Architecture:** 发布仅修改 changelog、release 文档和 Task 156；在隔离 worktree 中以 `VERSION=0.1.2` 执行完整 release-check 和 runtime version 验证。发布提交先合并并推送到 `origin/master`，确认同步后再经过单独人工批准创建 annotated tag。

**Tech Stack:** Git、Make、Go 1.22、Bash、curl、WSL `Ubuntu-24.04`。

## Global Constraints

- 版本为 `0.1.2`，tag 为 `v0.1.2`，发布日期为 `2026-07-12`。
- 不修改生产代码、外部 API、provider 行为或配置契约。
- 不构建或推送 registry 镜像，不创建托管平台 Release artifact。
- 最终验证必须在 WSL `Ubuntu-24.04` 中执行。
- tag 只能指向已推送且与 `origin/master` 同步的发布提交。
- 创建和推送 tag 前必须再次获得用户明确批准；此前的设计、计划和实施批准不授权 tag 操作。
